Hecklr

The Group Watch Platform by Jason Morris

  • Abstract

    A service that allows the group watch of content such as Youtube videos. Hecklr allows large group of viewers to be able to sync video playback for all. Complete with chat integration and on screen doodling, allowing great moments of humor as they group watch their favorite videos.

  • High Level Concept

    Using Youtube's API and Websocket to sync, Hecklr will let a group of users to coordinate syncing a video. Once a video is synced an started, real time chat features and on screen drawing will let the users express themselves as they watch the video together.

  • User Experience | Mechanics

    Youtube's API is capable of loading the same video for each user in the room. Once the video is loaded, each user will ready up. Once a certain percentage of users in the room have readied, the server will send the command for each client to play the video at the same time. If a new user joins during the middle of a viewing, the server will get the current playback time and send the current time to the new users so their video will match the other users' playback.

    During a playback, every user can interact in two ways, real time chat and on screen drawing. For chatting, a message will be sent to the server that will get dispatched to all the other users in the room. The drawing function will let users draw crud line art on top of the video playback for all to see, so you can give all your favorite Youtube celebs fancy moustaches and write funny thought bubbles for how they really feel.

  • Client | Server Breakdown

    The Client will be web based and use JavaScript and HTML for loading videos, interacting with the API, and the server/client coding.

    The Server will use Websockets and JavaScript while running on NodeJS. The server will group the users into rooms where once they are added they will get the current video/Youtube link from the Server's video queue. The server will then wait for ready ups from each user. Once enough readys are reach, the server will send the OK to each client and they will all start the video (weather they want to or not). If any new users join, the server will send the current video time (which it's been keeping track of) so that the new user can hop right in where everyone else is.

    The Server will also handle the passing of chat messages and the Canvas element that will used for the on screen drawing to everyone in the room so they can see what each other is saying and drawing.
